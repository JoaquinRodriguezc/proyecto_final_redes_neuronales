# `utils.py`

## Proposito del archivo

`utils.py` existe como placeholder para helpers reutilizables futuros.

## Estado actual

- El archivo esta vacio.
- No contiene funciones, clases ni constantes.
- No participa en el pipeline actual.

## Cuando tendria sentido usarlo

Solo conviene mover logica a `utils.py` si aparece un conjunto de helpers verdaderamente compartidos entre notebooks, scripts o modulos de `prod/`.

## Relacion con el proyecto actual

La implementacion reusable vigente ya esta separada en modulos mas especificos dentro de `prod/`, por lo que hoy no hay una necesidad concreta de concentrar helpers en este archivo.
